例2

HDU 5696 区间的价值

很经典的一道题目，之前也在校内模拟赛中见过，然而现在还不是很会

题解

对于当前处理的这个区间，我们可以找出这个区间的最小值和最大值。设最小值位置为qqq，最大值位置为ppp
因为只包含最小值是不可能成为答案的，所以我们用aq×apa_q \times a_paq​×ap​去更新区间∣p−q∣|p-q|∣p−q∣ ~ r−lr-lr−l

如果我们从中间分开，要讨论的情况比较多。
注意数据随机这一条件，因为数据随机，我们不需要从中间分治也可以使时间复杂度为O(nlog⁡n)O(n \log n)O(nlogn)
所以我们可以考虑从最小值处分治

接着递归[l,q)[l,q)[l,q)和[q+1,r)[q + 1,r)[q+1,r)

最后，我们发现随着长度的递减, 答案是单调递增的，所以每次只需要更新长度为r−lr−lr−l的答案，做一个后缀最大值即可

例3

UVa1608 Non-boring sequences

定义一个数列是合法的，当且仅当这个数列的每一个子串都至少存 在一个只出现过一次的元素。给定一个序列a1,a2,...,ana_1,a_2,...,a_na1​,a2​,...,an​，判断这个序列是否合法n≤2×105n \leq 2 \times 10^5n≤2×105

题解

老师说线段树做法很显然，可我不会啊。
然后老师讲了分治的做法

首先预处理出pre(i)pre(i)pre(i)和next(i)next(i)next(i)，分别表示与aia_iai​值相同的上一个位置和下一个位置。然后我们就能知道aia_iai​是否在[l,r][l,r][l,r]中只出现了一次。

设在这个区间只出现了一次的数的位置为ppp，显然，我们要递归[l,p)[l,p)[l,p)和[p+1,r)[p + 1,r)[p+1,r)。
但这道题的数据并不随机，我们这么递归很可能会使时间复杂度退化成O(n2)O(n^2)O(n2)。

我们可以这样分治：从左右端点同时开始找 只出现一次的数（只出现一次的数在大部分情况下不止一个），只要找到一个就进行递归
这么分治，时间复杂度为O(nlog⁡n)O(n \log n)O(nlogn)

堆

例1

给出一支股票，它在第iii天的价格为aia_iai​。每一天你可以花费aia_iai​买 入一股或者卖出一股得到aia_iai​的收益，可以什么都不做。求最大收益。
n≤105n ≤ 10^5n≤105

题解

用堆来贪心

到第iii天，如果ai<topa_i < topai​<top（top是堆顶）,那么ans+ai−topans + a_i - topans+ai​−top，弹出堆顶。
相当于在之前价格为堆顶时买入，第iii天卖出
接下来是最关键的一步：aia_iai​要入堆两次

为什么要入堆两次呢？

这是为了有一个“后悔”操作
如果只入堆一次，有可能会出现这种情况

如果只入堆一次，2,32,32,3匹配，3,43,43,4匹配，333被用了两次，显然不合法
如果不入堆，那么2,32,32,3匹配，1,41,41,4匹配
而最优的方案显然是2,42,42,4匹配
如果我们将333入堆两次，就不会出现以上情况

例2

给出一支股票，它在第iii 天的价格为aia_iai​ 。每一天你可以花费aia_iai​买
入一股或者卖出一股得到ai​a_i ​ai​​的收益，每天必须进行买卖。
求最大收益。

例1的变式，变得复杂了些。

题解

同样用一个小根堆维护所有已经卖出的节点。
我们每天的决策分为以下几步：

  1. 如果当前有一个之前买入的、未被匹配的点，卖出当前点，并将当前点加入堆以供未来的某一天抢走这个匹配。
  2. 如果当前不存在任何可以抢走匹配的点（或是还没有买任何点），只能买入。
  3. 如果堆顶大于当前股票价格，抢走堆中的匹配不能让答案变优，于是买入。
  4. 如果堆顶小于当前股票价格，就抢走当前堆顶的点的匹配（把堆顶变成买入，把新的卖出点加入堆），并记录接下来的某一天可以卖出一次（因为堆顶变成了买入）

并查集

例题

给出一个n×nn × nn×n的网格，每次删除其中的一条边，并且询问这条边被删除后它的两个端点是否连通（每次询问不独立）。强制在线。

题解

如果不强制在线的话，我们可以正难则反，把删边变成倒着加边
但是这道题强制在线。

所以就有了一个神仙做法：
建出对偶图，每删除一条边，就把相邻的两个格子并入一个集合。如果删除某条边后发现两个格子已经在一个集合内了，说明删除的边构成了一个闭合图形，两个点不连通。
用并查集维护即可

题目大意：

给出两个排列a,ba,ba,b，长度分别为 n,mn,mn,m，你需要计算有多少个xxx， 使得 a1+x,a2+x,...an+xa_1 + x,a_2 + x,...a_n + xa1​+x,a2​+x,...an​+x 是 bbb 的子序列

题解

因为我们要比较区间，先将aaa排列 hash
注意，因为aaa是一个排列，所以它里面的元素一定是从111~nnn的，所以，我们可以使pos[b[i]] = i，然后判断pos[i-n+1]~pos[i]和aaa排列是否符合要求
这里的符合要求包括：

  1. pos[i-n+1]~pos[i]减去xxx等于aaa排列
  2. b[pos[i-n+1]]~b[pos[i]]是bbb的子序列

而这两个要求都可以通过计算b[pos[i-n+1]]~b[pos[i]]在原排列里构成的子序列的hash值来判断
之后我们要做的就是快速的求出b[pos[i-n+1]]~b[pos[i]]在原排列bbb中构成的子序列的hash值
