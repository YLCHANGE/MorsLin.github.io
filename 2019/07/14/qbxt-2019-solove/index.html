题目是二维问题，不好处理，可以考虑转化成一维问题

最终，此题可以转化为经典的区间选点问题

例5

Noip1999 旅行家的预算

加的油肯定越便宜越好，每到一个加油站，我们只加最低限度的油，即加的油刚好够跑到下一个比当前便宜的加油站
如果在当前加油站加满油都跑不到下一个加油站，那么无解
还需要注意的一点是，如果后面没有更加便宜的加油站，那么就要在当前加油站加满油（能多便宜一会儿是一会儿）

例6

POI2010 GRA-The Minima Game

显然，我们要让自己取的数中最小值尽量大，还要使剩下的数中最大值尽量小
所以先手一定会从大到小取数，且取出的数为从大到小的连续的一段，因此我们先把所有数从小到大排序
接下来就要用一种类似DPDPDP的思想，设f[i]表示两个人从111取到iii时，分值之差的最大值
对于每一个iii，我们还要枚举AAA取到哪里，设AAA从iii取到jjj，则f[i] = max(f[i], a[j] - f[j-1])，即我们要从所有的a[j] - f[j-1]中取一个最大值
因为f[i-1]是从子问题的最大值转移过来的，即f[j]必定小于等于f[i-1]，所以f[i-1]就是最大值
最终，转移方程为f[i] = a[i] - f[i-1]

例4

USACO2008JAN 猜数游戏
原题在BZOJ上为权限题，题目大意如下

有一个长度为nnn的未知数列和QQQ条限制li,ri,xil_i,r_i,x_ili​,ri​,xi​，表示数列l∼rl∼rl∼r间所有数的最小值为xxx

n≤1000000,Q≤25000n \leq 1000000, Q \leq 25000n≤1000000,Q≤25000

题解

显然只有当有两个区间产生包含关系且被包含的区间最小值更小才会出现矛盾

Noip2015 运输计划

用LCALCALCA求出所有路径的总时间，然后二分答案ttt，则小于等于ttt的路径可以舍去
对于大于ttt的路径，我们必须要找到一条它们都经过的且最大的边，将这条边变为虫洞。如果 不存在它们都经过的边 或 最久的计划−-−此边边权$ > t$则无解，反之有解

接下来就是如何判断被所有时间大于ttt的计划经过的边
可以使用树上差分
对于一条从uuu到vvv的计划，将点u,vu,vu,v的点权+1+1+1，u,vu,vu,v的LCALCALCA点权−2-2−2，对于一个点求它的子树和，若子树和 === 所有$ > t $的计划的数量，则此点和它的父节点之间的边即为所求

例2

Usaco2014 Mar The Lazy Cow

BZOJ和Luogu都有此题，但BZOJ上是权限题，Luogu上数据范围过水，数据范围可以达到1≤N≤100000,1≤K≤20000001 \leq N \leq 100000, 1 \leq K \leq 20000001≤N≤100000,1≤K≤2000000

题解

此题为曼哈顿距离，范围为一个菱形，我们可以将坐标系旋转π4\frac{ \pi }{4}4π​，使其变为正方形
然后此题变为正方形所包含的区间的最大区间和

可以使用扫描线，用线段树维护
对yyy建线段树，枚举xxx，若枚举xxx时，在yyy上存在一点作为正方形的左下角，刚好可以使一堆草在正方形的右上角，则线段树将此段区间加上草的价值，等无法覆盖此点时再减去
扫描过程中，求线段树的最大值即可

例3

LA7141 BombX
题意

无穷大的地图上有n个兵，现给出一个W*H的炸弹（不可以旋转），一次能炸死它所在的行和列的所有小兵，炸弹不能放在小兵的上面。
问：一次最多能干掉多少个小兵？

1≤n≤105,0≤W,H,x,y≤1071 \le n \le 10 ^ 5, 0 \le W,H,x,y \le 10 ^ 71≤n≤105,0≤W,H,x,y≤107

题解

同样是扫描线，只不过我们这次还要判断炸弹所在的位置，还是炸弹的左下角，如果放不了，就在线段树中加一个极小值，使此区间无法成为答案

例4

Hdu4630 No Pain No Game

题意

给出一个1 n1~n1 n的排列。QQQ个询问，询问区间[l,r][l, r][l,r]间gcd⁡(ai,aj)\gcd{(ai,aj)}gcd(ai,aj)的最大值。
l=rl = rl=r时输出000
1≤n,Q≤500001 \le n,Q \le 500001≤n,Q≤50000

题解

将询问按右端点从小到大排序，处理到一个数字xxx时，对于xxx的一个因子ddd，找到上一次出现因子ddd的位置pospospos，则之后，所有左端点小于pospospos的询问都可以以ddd为答案
用线段树维护，对pospospos到xxx所在的位置这段区间的数和ddd之间取最大值
